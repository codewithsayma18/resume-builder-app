Generate a professional IT job resume in JSON format based on the following description. 
Ensure the JSON is consistent, well-structured, and contains all specified keys. 
If information is missing for a specific field, use null or an empty string/list as appropriate, but do not remove the key.

Input Description:
"{{userDescription}}"

JSON Structure Requirements:

1. personalInformation:
   - fullName (string)
   - email (string)
   - phoneNumber (string)
   - location (string)
   - linkedIn (string or null)
   - gitHub (string or null)
   - portfolio (string or null)

2. summary: A brief professional overview of skills, experience, and career goals (string).

3. skills: A list of objects containing:
   - title (string)
   - level (string, e.g., "Beginner", "Intermediate", "Advanced")

4. experience: A list of job roles. Each object must include:
   - jobTitle (string)
   - company (string)
   - location (string)
   - duration (string, e.g., "Jan 2020 - Present")
   - responsibility (string)  <-- Summarize responsibilities in one paragraph or bullet points separated by newlines.

5. education: A list of degrees. Each object must include:
   - degree (string)
   - university (string)
   - location (string)
   - graduationYear (string)

6. certifications: A list of certifications. Each object must include:
   - title (string)
   - issuingOrganization (string)
   - year (string)

7. projects: A list of key projects. Each object must include:
   - title (string)
   - description (string)
   - technologiesUsed (array of strings)
   - githubLink (string or null)

8. achievements: A list of objects containing:
   - title (string)
   - year (string)
   - extraInformation (string)

9. languages: A list of objects containing:
   - id (number, auto-increment starting from 1)
   - name (string)

10. interests: A list of objects containing:
    - id (number, auto-increment starting from 1)
    - name (string)

Response Format:
Return ONLY the JSON object. Do not add markdown formatting (like ```json ... ```) if possible.
